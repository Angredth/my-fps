## 오늘 Codex로 처리한 작업

### 1. ER Fulcrum 근접무기 CDN import 적용

- `5.28/index/SKOV.html`의 근접무기 로딩 경로를 로컬 `./er-fulcrum.js`에서 GitHub 커밋 고정 CDN으로 변경했다.
- 적용한 CDN:

```js
https://cdn.jsdelivr.net/gh/Angredth/my-fps@fc72912d1c39bd6126cbea727d07535e7994c26c/5.28/index/er-fulcrum.js
```

- `createTacticalBayonetModel` 또는 default export를 사용하도록 기존 fallback 구조를 유지했다.
- CDN 응답 확인 결과:
  - `HTTP 200`
  - `content-type: application/javascript`
  - `access-control-allow-origin: *`

### 2. 탈출 결과음 `Exit.mp3` 재생 문제 분석 및 보강

- `Exit.mp3` 파일 자체는 정상으로 확인했다.
  - 루트 `Exit.mp3`
  - `5.28/index/Exit.mp3`
  - 둘 다 `audio/mpeg`로 서버 응답 확인

- 탈출 흐름은 다음 구조로 확인했다.

```text
completeExtraction()
-> updateRaidEndFade()
-> InventoryBridge.finishRaidExtraction()
-> playRaidOutcomeSound('extract')
```

- 기존 문제:
  - `audio.play()` 실패 시 실패 원인을 거의 숨기고 곧바로 완료 처리로 넘어갔다.
  - 첫 오디오 소스가 실패하면 fallback 소스를 실질적으로 다시 재생 시도하지 못하는 구조였다.
  - 그래서 탈출 처리는 진행되지만 결과음 로직이 기능하지 않는 것처럼 보였다.

- 보강 내용:
  - 결과음 재생을 소스별로 순차 시도하도록 변경했다.
  - 실패 시 다음 fallback 경로로 넘어가도록 했다.
  - `NotAllowedError`, 로딩 실패 등 재생 실패 원인을 콘솔에 남기도록 했다.
  - 결과음 재생 중에는 메뉴 음악이 끼어들지 않도록 기존 blocking 상태를 유지했다.

### 3. 근접무기 위치 조절 이슈 원인 확인

- `createKnifeModel()` 안의 아래 값은 이후 코드에서 덮어써진다.

```js
fulcrum.position.set(0.42, -0.12, 1);
```

- 이유:
  - `createKnifeModel()`이 반환한 `fulcrum` 객체를 아래에서 `knife`로 받고,
  - 다시 `knife.position.set(...)`을 호출하기 때문이다.

```js
const knife = createKnifeModel();
knife.position.set(0.25, -0.12, 0.2);
```

- 따라서 현재 1인칭 근접무기 장착 위치를 조절하려면 우선 `knife.position.set(...)`을 조절해야 한다.
- 모든 무기의 1인칭 들고 있는 위치는 매 프레임 `weaponGroup.position`과 `targetLocalPos` 계산이 지배한다.

## 참고

- 현재 `SKOV.html`은 GitHub CDN에서 ER Fulcrum 모델을 가져오므로, 로컬 `er-fulcrum.js`를 수정해도 근접무기 원본 모델에는 반영되지 않는다.
