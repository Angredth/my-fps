## 2. 버튼 및 ESC UI 효과음 보강

- 버튼 hover/click 효과음 재생 로직을 보강했다.
- `Esc.mp3`는 ESC 키 입력 자체가 아니라, ESC로 실제 동작이 처리된 경우에만 재생되도록 변경했다.
- ESC로 처리되는 동작:
  - 컨테이너 창 닫기
  - 레이드 메뉴 열기/닫기
  - 루팅/인벤토리 오버레이 닫기
  - 인벤토리/상인 화면에서 메뉴로 복귀
- `Esc.mp3` 앞부분 무음 구간을 줄이기 위해 시작 오프셋을 적용했다.

```js
escape: 0.18
```

## 5. 공통 설정 시스템 추가

- 메인 화면 우측 하단에 톱니바퀴 설정 버튼을 추가했다.
- `settingsModal` 기반 공통 설정창을 추가했다.
- 현재 구현 탭은 `SOUND`이며, 이후 `graphics`, `controls`, `game` 확장을 고려한 구조로 작성했다.
- 설정창 구성:
  - `SETTINGS` 제목
  - `SOUND` 탭
  - 메인 볼륨
  - 인터페이스 볼륨
  - 음악 볼륨
  - `RESET`, `REVERT`, `SAVE`
- 설정창은 바깥 오버레이 클릭 또는 ESC로 닫을 수 있다.

## 6. 설정 저장 구조 추가

- `localStorage` 저장 키를 추가했다.

```js
gameSettings
```

- 기본 설정 구조:

```js
{
  version: 1,
  sound: {
    master: 100,
    fx: 100,
    music: 50
  },
  graphics: {},
  controls: {},
  game: {}
}
```

- `savedSettings`와 `draftSettings`를 분리했다.
  - `savedSettings`: 실제 저장 및 적용된 설정
  - `draftSettings`: 설정창에서 수정 중인 임시 설정
- `SAVE` 전에는 `localStorage`에 반영하지 않는다.

## 7. AudioManager 추가

- 공통 볼륨 계산을 위한 `AudioManager`를 추가했다.
- 카테고리:
  - `interface`: 버튼, ESC 등 UI 효과음
  - `music`: 메인 메뉴 음악
  - `gameplay`: 총기, 발소리, 피격, 환경음, 탈출/사망 결과음
- 볼륨 계산:

```js
interface = baseVolume * master * fx
music = baseVolume * master * music
gameplay = baseVolume * master
```

- 총기, 발소리, 피격음, 환경음에는 `fx`가 적용되지 않도록 분리했다.
- 현재 재생 중인 메뉴 음악은 설정 변경 시 즉시 볼륨이 갱신되도록 했다.

## 8. 설정 버튼 표시 조건

- 설정 버튼은 메인 화면에서만 표시되도록 했다.
- 레이드 진입 시 설정창이 열려 있으면 저장하지 않은 변경사항을 폐기하고 닫는다.
- 레이드 진행 중에는 설정 버튼을 숨긴다.
- 레이드 종료 후 메인 화면으로 돌아오면 설정 버튼이 다시 표시된다.

## 9. 런타임 오류 수정

- `Esc.mp3` 효과음을 추가하면서 `buttonSfx`에는 `escape`가 추가됐지만, `buttonSfxPool`과 `buttonSfxPoolIndex`에는 누락되어 페이지 초기화가 중단되는 문제가 있었다.
- 이로 인해 메인 버튼과 메인 음악 재생 흐름이 멈출 수 있었다.
- 수정 내용:

```js
const buttonSfxPool = { hover: [], click: [], escape: [] };
const buttonSfxPoolIndex = { hover: 0, click: 0, escape: 0 };
```
