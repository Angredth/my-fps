# 6.15 Patch

작성일: 2026-06-15

## 오늘 Codex로 처리한 작업

### 1. 시체 수색 상호작용 판정 변경

- `5.28/index/SKOV.html`의 시체 수색 상호작용에서 근거리 fallback 판정을 제거했다.
- 기존에는 화면 중앙 레이캐스트가 시체에 맞지 않아도 가까운 시체가 있으면 상호작용할 수 있었다.
- 변경 후에는 화면 중앙 레이캐스트가 시체 ragdoll mesh에 직접 맞아야만 상호작용된다.
- 시체 상호작용 거리는 더 가까운 수색감을 위해 `2.0m`로 조정했다.

```js
raycaster.far = 2.0;
```

### 2. 시체 수색 표시 UI 변경

- 기존 `F · 시체 수색` 단일 텍스트 UI를 중앙점 기준 2단 라벨 구조로 변경했다.
- 중앙점 위에는 현재 행동을 표시한다.
- 중앙점 아래에는 현재 상호작용 대상을 표시한다.

```html
<span class="interaction-label interaction-action">수색</span>
<span class="interaction-label interaction-target">시체</span>
```

- `수색` 라벨:
  - 밝은 회색 채움
  - 검은 글씨
  - 검은 윤곽선
  - 각진 사각형

- `시체` 라벨:
  - 검은색 채움
  - 흰 글씨
  - 윤곽선 없음
  - 각진 사각형

- 두 라벨 공통 크기를 약 30% 축소했다.

```css
min-width: 38px;
padding: 3px 7px;
font-size: 10px;
```

### 3. 월드 루팅 모델 CDN 로딩 추가

- `SKOV.html`의 FPS 모듈에서 월드 루팅 모델을 GitHub jsDelivr CDN으로 불러오도록 추가했다.
- 적용한 CDN base:

```js
https://cdn.jsdelivr.net/gh/Angredth/my-fps@main/5.28/index/
```

- 적용 모델:
  - `Comp.js`
  - `WoodenCrate.js`
  - `Toolbox.js`
  - `SupplyCrate.js`

- 일부 모델 파일은 ES module export 형식이 아니므로, CDN 소스를 `fetch()`로 읽어 factory 함수를 추출해 사용하는 구조로 처리했다.
- 모델 로드 또는 생성 실패 시 임시 박스 모델로 fallback한다.

### 4. 건물 옆 월드 루팅 오브젝트 배치

- 인게임 건물 구조물 옆에 루팅 모델을 건물당 랜덤 1개씩 배치하도록 추가했다.
- 지역 경계벽에는 배치하지 않고, 기존 건물 생성 루프에서만 배치한다.
- 건물 네 면 중 하나를 랜덤으로 골라 외곽에 배치한다.
- 지형 높이에 맞춰 모델 바닥이 땅에 닿도록 위치를 보정한다.
- 스폰 지점, 탈출구 등 주요 지점 근처에는 배치하지 않도록 기존 검증 함수를 사용했다.

### 5. 월드 루팅 모델별 테이블 매핑

- 월드 루팅 오브젝트는 기존 루팅 테이블을 그대로 사용한다.

```text
comp         -> computer
woodencrate -> weapon
toolbox      -> toolbox
supplycrate  -> medical
```

- 표시 대상명:
  - `컴퓨터`
  - `무기 상자`
  - `공구 상자`
  - `의료품 박스`

### 6. 월드 루팅 모델 스케일 적용

- 요청에 따라 `Comp.js`와 `WoodenCrate.js`는 `1/100` 스케일로 배치했다.

```js
scale: 0.01
```

- `Toolbox.js`는 1:1로 배치한다.
- `SupplyCrate.js`는 모델 내부 단위가 `cm`로 선언되어 있어 게임 월드의 미터 단위에 맞게 단위 변환용 `0.01`만 적용했다.

### 7. 월드 루팅 상호작용 추가

- 월드 루팅 오브젝트도 `F` 키로 상호작용하도록 추가했다.
- 시체와 동일하게 화면 중앙 레이캐스트가 대상 모델에 직접 맞고 `2.0m` 이내일 때만 상호작용된다.
- 기존 상호작용 UI를 재사용하며, 대상명만 동적으로 변경된다.

예시:

```text
수색
중앙점
컴퓨터
```

- `F` 입력 시 대상 종류에 따라 분기한다.
  - 시체: 기존 `openEnemyCorpse()`
  - 월드 루팅: 신규 `openWorldLootContainer()`

### 8. 월드 루팅 수색 브리지 추가

- 기존 POI 수색 흐름을 일반화해서 월드 루팅 오브젝트도 같은 인벤토리 오버레이와 루팅 컨테이너를 사용하도록 변경했다.
- 추가된 흐름:

```text
openWorldLootContainer()
-> startSearchTarget()
-> updateSearchProgress()
-> completeSearch()
-> generateLootContainer()
```

- 기존 POI 버튼 수색은 유지된다.
- 이미 수색한 월드 루팅 컨테이너는 같은 key로 저장되어 재상호작용 시 기존 컨테이너를 다시 연다.

### 9. 검증

- `SKOV.html` 일반 인라인 스크립트 문법 검사 통과
- `SKOV.html` module script 문법 검사 통과

```bash
node --check /tmp/skov-inventory-script.js
node --check /tmp/skov-module.js
```

## 참고

- 월드 루팅 모델은 CDN에서 불러오기 때문에 네트워크가 막히면 fallback 박스 모델이 표시된다.
- 로컬 정적 서버는 `8000`번 포트로 실행했지만, 현재 셸에서 `curl` 포트 확인은 실패했다.
- 실제 브라우저에서 확인할 핵심 항목:
  - 건물 옆 루팅 모델 생성 여부
  - `Comp.js`, `WoodenCrate.js` 스케일
  - 중앙점 조준 시 상호작용 UI 표시
  - `F`로 월드 루팅 수색 오버레이 열림
  - 수색 완료 후 기존 루팅 테이블 아이템 생성
