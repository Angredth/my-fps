## 오늘 Codex로 처리한 작업

### 1. 인벤토리 아이템 카드 UI 1단계 변경

- `5.28/index/SKOV.html`의 `renderGrid()`가 생성하는 그리드 아이템 표시 방식을 타르코프형 카드 구조로 변경했다.
- 기존처럼 아이템 이름이 카드 중앙에 크게 표시되던 방식을 제거했다.
- 카드 내부 표시 구조를 DOM API 기반 생성 함수로 분리했다.

```js
function createItemCardContents(item) {
  const fragment = document.createDocumentFragment();
  ...
  fragment.append(header, visual, footer);
  return fragment;
}
```

- `renderGrid()`에서는 기존 아이템 노드의 좌표, 크기, 클래스, 이벤트는 유지하고 내부 표시 콘텐츠만 교체한다.

```js
node.replaceChildren(createItemCardContents(item));
```

### 2. 아이템 약칭 코드 표시 추가

- 아이템 카드 우측 상단에 짧은 코드가 표시되도록 추가했다.
- `shortName`이 있는 아이템은 해당 값을 사용한다.
- `shortName`이 없으면 아이템 이름 또는 id에서 안전한 대체 코드를 생성한다.

```js
function getItemShortName(item) {
  if (item.shortName) return item.shortName;
  const source = String(item.name || item.id || 'ITEM').trim();
  if (!source) return 'ITEM';
  return source.replace(/\s+/g, '').slice(0, 5).toUpperCase();
}
```

- 일부 `ITEM_CATALOG` 항목에 표시 전용 `shortName`만 추가했다.
- 아이템 기능 데이터, 경제 데이터, 저장 구조는 변경하지 않았다.

### 3. 좌측 상단 아이템 이름 표시 제거

- 최초 카드 구조에는 좌측 상단 이름과 우측 상단 코드가 함께 표시되었으나, 추가 요청에 따라 좌측 상단 이름 표시를 제거했다.
- 현재 카드 헤더에는 우측 상단 코드만 표시된다.
- 전체 아이템 이름은 기존 `title` 툴팁에서 계속 확인할 수 있다.

```js
header.append(code);
```

### 4. 아이콘 및 플레이스홀더 구조 추가

- `item.icon`이 있을 때만 중앙 이미지 요소를 생성한다.
- 이번 패치에서는 실제 외부 이미지 경로를 임의로 추가하지 않았다.
- 아이콘이 없으면 중앙에 흐린 약칭 플레이스홀더를 표시한다.
- 이미지 로드 실패 시 이미지 요소를 숨기고 플레이스홀더를 다시 표시한다.

```js
image.addEventListener('error', () => {
  image.hidden = true;
  placeholder.hidden = false;
});
```

### 5. 카드 하단 상태 표시 구조 추가

- 카드 하단에는 기존 데이터 기반으로만 상태 텍스트를 표시하도록 했다.
- 표시 대상:
  - 내구도
  - 장탄 상태
  - 회전 상태
  - 컨테이너 여부

- 실제 값이 없는 아이템은 빈 문자열로 유지한다.
- 신규 수량 데이터나 스택 데이터는 추가하지 않았다.

### 6. 1x1 아이템 레이아웃 대응

- 1x1 아이템에는 별도 클래스가 붙도록 했다.

```js
node.classList.toggle('item-card--one-cell', size.width === 1 && size.height === 1);
```

- 1x1 카드에서는 이름 표시를 하지 않고 우측 상단 코드와 중앙 플레이스홀더 중심으로 표시한다.
- 텍스트가 카드 밖으로 넘치지 않도록 코드와 플레이스홀더 크기를 제한했다.

### 7. 상점 가격 및 재고 태그 위치 조정

- 상점 아이템의 기존 가격과 재고 데이터는 유지했다.
- `.price-tag`는 좌측 하단, `.stock-tag`는 우측 하단으로 배치했다.
- 상점 아이템에서는 일반 카드 하단 상태 표시를 숨겨 가격/재고와 겹치지 않도록 했다.


- 재고 정보는 기존 `title` 툴팁 문자열에 유지된다.

```css

  display: none;
}
```

### 12. 범용 스택 데이터 구조 2단계 추가

- 이후 루블, 탄약, 의료품 등에 공통으로 사용할 수 있는 범용 스택 데이터 기반을 추가했다.
- 이번 단계에서 실제 스택 병합, 분할, 자동 이동, 상점 화폐 전환은 구현하지 않았다.
- 현재 스택 가능 아이템은 루블 하나만 사용한다.

```js
stackable
stackKey
quantity
maxStack
```

- 루블 템플릿에 스택 필드를 추가했다.

```js
{
  id: 'ruble',
  name: '루블',
  stackable: true,
  stackKey: 'ruble',
  quantity: 1,
  maxStack: 500000
}
```

- 범용 스택 유틸리티 함수를 추가했다.

```js
function isStackableItem(item) { ... }
function getStackLimit(item) { ... }
function getStackQuantity(item) { ... }
function canStackItems(firstItem, secondItem) { ... }
```

- `cloneItem()`에서 스택 아이템 생성 시 독립적인 `quantity`, `stackKey`, `maxStack`을 보정하도록 했다.
- 신규 루블은 `quantity: 1`, `maxStack: 500000`을 가진 독립 아이템으로 생성된다.
- 루블을 여러 번 생성해도 기존처럼 `uid`는 각각 다르게 생성된다.

### 13. 기존 저장 데이터 스택 호환 처리

- `normalizeItem()`에서 기존 저장 데이터의 루블 아이템도 스택 필드를 복구하도록 했다.
- 수량 없는 기존 루블 저장 데이터는 로드 시 다음 값으로 정규화된다.

```text
stackable = true
stackKey = ruble
quantity = 1
maxStack = 500000
```

- 기존 `uid`, `x`, `y` 좌표는 유지한다.
- `containerItems.map(normalizeItem)` 재귀 정규화 흐름은 유지해서 지갑 등 컨테이너 내부 루블도 같은 방식으로 복구된다.
- 저장된 개별 루블 수량이 `500000`을 넘으면 해당 아이템의 `quantity`만 `500000`으로 제한한다.
- 저장 키, 저장 버전, `state.money` 마이그레이션은 추가하지 않았다.

### 14. 스택 수량 표시 추가

- `itemInfo()`는 스택 아이템에만 수량 정보를 표시하도록 변경했다.

```text
루블 · 수량 1 / 500,000 · 1x1 · ₽1 · 0.01kg · 기본 화폐
```

- 일반 아이템에는 수량 정보를 표시하지 않는다.

```text
붕대 · 1x1 · ₽3,000 · 0.2kg · 회복 아이템
```

- `renderGrid()`에서 스택 아이템에만 `.item-stack-count` 배지를 추가했다.

```js
if (isStackableItem(item)) {
  const stackCount = document.createElement('span');
  stackCount.className = 'item-stack-count';
  stackCount.textContent = getStackQuantity(item).toLocaleString('ko-KR');
  node.appendChild(stackCount);
}
```

- 일반 아이템에는 `quantity`, `stackKey`, `maxStack`을 임의로 추가하지 않는다.
- 수량 표시 여부는 `quantity` 존재 여부가 아니라 `isStackableItem(item)`으로만 판정한다.
