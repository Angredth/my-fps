# 6.18 Patch

작성일: 2026-06-18

## 오늘 Codex로 처리한 작업(8단계 완료)

### 1. 레이드 종료 시 사운드 정리

- 플레이어 사망 또는 탈출 처리에 진입하면 레이드 진행 중 소리가 즉시 멈추도록 정리했다.
- 사망 처리 함수와 탈출 처리 함수에서 공통으로 레이드 사운드 정지 함수를 호출한다.
- 적 사격음, 환경음 등 `gameplay` 계열로 재생 중인 사운드가 레이드 종료 후 남지 않도록 했다.

```js
function stopRaidSounds() {
  ...
}
```

- 적용 위치:
  - `finishPlayerDeath()`
  - `finishRaidExtraction()`
  - 레이드 종료/세션 종료 흐름

### 2. 루블 스택 병합 기능 추가

- 루블 아이템을 스택형 아이템으로 다루기 위한 공통 유틸을 정리했다.
- 추가 및 활용한 주요 함수:

```js
isStackableItem(item)
getStackQuantity(item)
getStackLimit(item)
canStackItems(firstItem, secondItem)
mergeStackItems(sourceItem, targetItem)
removeItemByUidFromList(items, uid)
```

- 동일한 루블 스택을 다른 루블 스택 위에 드롭하면 대상 스택의 최대치까지 병합된다.
- 대상 스택 최대치는 `500,000`이다.
- 초과 수량은 원본 스택에 그대로 남는다.
- 원본 수량이 0이 되면 원본 아이템을 UID 기준으로 정확히 제거한다.

예시:

```text
300,000 + 100,000 -> 400,000
480,000 + 100,000 -> 500,000 + 80,000
500,000 + 10,000 -> 변화 없음
```

- 아이템 위 드롭 처리에서는 동일 스택 병합을 먼저 시도하고, 병합 조건이 아니면 기존 소음기 장착, 컨테이너 삽입, 기타 드롭 로직을 유지한다.

### 3. Ctrl+좌클릭 빠른 이동 스택 자동 병합

- `quickMoveToAreas()` 흐름에서 빈칸 탐색 전에 목적지 영역의 병합 가능한 루블 스택을 먼저 찾도록 했다.
- 병합 가능한 스택은 수량이 큰 스택부터 채운다.

```js
getMergeableStacksInArea(item, area)
tryQuickMergeStackToArea(from, uid, to, options)
mergeStackQuantity(source, target)
```

- 빠른 이동 시 일부만 병합 가능한 경우에도 아이템 손실 없이 원본 수량을 유지한다.
- 병합 후 남은 수량이 있고 목적지에 빈칸이 있으면 기존 이동 흐름으로 이어진다.
- 저장과 렌더링이 중복 호출되지 않도록 `silent` 옵션을 연결했다.

### 4. Ctrl+드래그 스택 분할 팝업 추가

- `Ctrl+드래그` 후 빈 그리드 칸에 드롭하면 스택 분할 팝업이 열리도록 추가했다.
- `Ctrl+좌클릭` 빠른 이동과 충돌하지 않도록 `dragend` 직후 클릭 억제 시간을 둔다.

```js
let modifierState = { ctrl: false };
let suppressItemClickUntil = 0;
let pendingStackSplit = null;
```

- 추가한 UI:
  - `#splitStackModal`
  - range 입력
  - number 입력
  - 분할 수량 / 남은 수량 표시
  - `ACCEPT` 버튼

- 팝업을 열 때는 원본 수량을 변경하지 않는다.
- `ACCEPT` 시점에만 원본 수량을 줄이고 새 UID를 가진 분할 스택을 생성한다.
- 취소, Escape, 닫기 버튼, 배경 클릭은 상태를 변경하지 않는다.
- 팝업이 열린 뒤 대상 칸이 차거나 원본 수량이 바뀌면 분할을 취소한다.

### 5. 루블 전용 판매 제한 규칙 추가

- 루블은 판매할 수 없도록 공통 판매 판정 함수를 추가했다.

```js
hasContainerContents(item)
getSellBlockReason(item)
canSellItem(item)
cleanupTradeSellItems()
```

- 적용된 판매 차단:
  - 루블 판매 금지
  - `sellAll()`에서 루블 제외
  - `Ctrl+좌클릭` 판매 등록에서 루블 차단
  - 판매 영역 드롭에서 루블 차단
  - 내용물이 있는 지갑, 백팩, 리그, 보안 컨테이너 판매 금지
  - `unsearched`, `searching` 컨테이너 판매 금지

- 실패 메시지:

```text
루블은 판매할 수 없습니다.
내용물이 있는 컨테이너는 판매할 수 없습니다.
```

- 상점 화면에서는 판매 불가 아이템에 `non-sellable` 클래스를 붙여 어둡게 표시한다.
- 판매 불가 아이템에는 `NO SELL` 라벨과 툴팁 사유가 표시된다.

### 6. 루블 무게 및 툴팁 표시 정리

- 루블은 수량과 무관하게 무게 계산에서 0kg으로 처리한다.

```js
function itemTotalWeight(item) {
  if (item.id === 'ruble') return 0;
  return Number(item.weight) || 0;
}
```

- 스택 아이템 툴팁에는 현재 수량과 최대 수량을 표시한다.

예시:

```text
루블 · 45,000/500,000 · 1x1 · ₽1 · 0kg · 기본 화폐
```

- 수량이 0 이하인 스택은 저장 전 정리되도록 방어 코드를 유지했다.

### 7. 실사용 장비 루블 200,000 제한

- 실사용 장비 전체에 보관 가능한 루블 총량을 `200,000`으로 제한했다.

```js
const PLAYER_USABLE_ROUBLE_LIMIT = 200000;
```

- 합산 대상:
  - 리그 내부
  - 주머니
  - 백팩 내부
  - 보안 컨테이너 내부
  - 위 공간 안에 들어 있는 지갑
  - 중첩 컨테이너 내부 루블

- 추가한 주요 함수:

```js
isRoubleItem(item)
getRoubleTotalInItems(items)
getUsableGearRoubleTotal()
isUsableGearArea(area)
getIncomingUsableRoubleAmount(item, sourceArea, targetArea, amountOverride)
canMoveRoublesIntoUsableGear(item, sourceArea, targetArea, amountOverride, options)
```

- 스태시나 루팅 테이블에서 실사용 장비로 루블을 옮길 때 한도를 넘으면 이동을 차단한다.
- 실사용 장비 내부 이동은 총량이 늘지 않으므로 허용한다.
- 지갑 안에 루블을 넣어 한도를 우회하는 것도 차단한다.
- 스택 병합, 빠른 이동, 일반 드래그 이동, 컨테이너 삽입, 장비 장착 경로에 같은 한도 검사를 연결했다.
- 분할 팝업은 대상이 실사용 장비일 경우 남은 한도만큼만 최대값이 잡힌다.

실패 메시지:

```text
실사용 장비에는 최대 200,000루블까지만 보관할 수 있습니다.
```

### 8. 저장 데이터 마이그레이션 추가

- 기존 `state.money` 숫자 자금을 실제 루블 스택으로 1회 변환하는 저장 데이터 마이그레이션을 추가했다.
- 저장 키는 변경하지 않았다.

```js
const SAVE_KEY = 'tarkov-style-web-prototype-state-v5';
const STATE_SCHEMA_VERSION = 6;
const RUBLE_STACK_LIMIT = 500000;
```

- 추가한 주요 함수:

```js
createRoubleStack(amount)
placeRoubleStacksInStash(stash, totalAmount)
removeLegacyRoubleItems(items)
sumRoublesInItems(items)
migrateMoneyToRoubleStacks(normalizedState, saved)
```

- 변환 규칙:

```text
45,000    -> 45,000
500,000   -> 500,000
1,200,000 -> 500,000 + 500,000 + 200,000
```

- `schemaVersion`이 6 이상이면 다시 변환하지 않는다.
- 기존에 수량 있는 루블 스택이 이미 `money` 이상 존재하면 중복 생성하지 않는다.
- 구형 `quantity` 없음 또는 `quantity <= 1` 루블은 마이그레이션 복사본에서 제거한다.
- 스태시 공간이 부족하면 원본 상태를 훼손하지 않고 `schemaVersion`도 올리지 않는다.
- `state.money`는 상점 호환을 위해 유지했다.
- 상점 구매/판매 경제는 아직 기존 `state.money` 기반으로 작동한다.

### 9. 마이그레이션 디버그 도우미 추가

- 브라우저 콘솔에서 저장 데이터 마이그레이션을 재현할 수 있도록 `window.DEBUG_MIGRATION`을 추가했다.

```js
DEBUG_MIGRATION.makeLegacySave(45000)
DEBUG_MIGRATION.inspectRoubles()
```

- `makeLegacySave(money)`는 구형 저장 데이터를 만들고 새로고침한다.
- `inspectRoubles()`는 현재 저장 데이터의 `schemaVersion`, `money`, 스태시 루블 총액을 출력한다.
