# 6.19 Patch

작성일: 2026-06-19

## 오늘 Codex로 처리한 작업

### 1. 상점 경제를 루블 스택 기반으로 전환

- 기존 숫자 자금 `state.money` 증감 방식에서 벗어나, 실제 인벤토리 안에 존재하는 `ruble` 스택 수량을 자금 원본으로 사용하도록 변경했다.
- 상단 보유 자금 표시는 `state.money`가 아니라 플레이어가 소유한 루블 아이템 총합을 표시한다.

```js
getOwnedRoubleTotal()
```

- 합산 대상:
  - 스태시
  - 스태시 내부 지갑 및 컨테이너
  - 주머니
  - 장착 리그 내부
  - 장착 백팩 내부
  - 보안 컨테이너 내부
  - 위 공간 안의 중첩 컨테이너

- 구매 시에는 실제 루블 스택에서 가격만큼 수량을 차감한다.
- 차감 순서는 작은 루블 스택 우선이다.
- 구매 아이템을 놓을 스태시 공간이 없으면 루블을 먼저 차감하지 않는다.
- 루블이 부족하면 구매는 실패하고 아이템도 생성되지 않는다.

### 2. 판매 수익을 루블 아이템으로 지급

- 판매 완료 시 `state.money += total` 방식 대신 스태시에 루블 스택을 생성하거나 기존 스택에 병합하도록 변경했다.

```js
canStoreRoublesInStash(total)
addRoublesToStash(total)
```

- 판매 수익 지급 규칙:
  - 기존 스태시 루블 스택 중 가장 큰 스택부터 채운다.
  - 한 스택은 최대 `500,000` 루블까지만 보관한다.
  - 남은 금액은 새 `1x1` 루블 스택으로 생성한다.
  - 수익을 보관할 공간이 없으면 판매 자체를 취소한다.
  - 판매 실패 시 판매 등록 아이템과 루블 수량은 유지된다.

예시:

```text
기존 루블 480,000 + 판매 수익 50,000
-> 기존 스택 500,000
-> 새 스택 30,000
```

### 3. 구형 숫자 자금 코드 최종 정리

- 런타임 거래 로직에서 `state.money` 직접 사용을 제거했다.
- 구매, 판매, 자금 표시, 자금 비교는 루블 아이템 합산 함수만 사용한다.
- `formatMoney()`는 숫자를 표시 문자열로 바꾸는 포맷팅 전용 함수로 정리했다.

```js
formatMoney(getOwnedRoubleTotal())
```

- 저장 데이터 정규화 결과에는 더 이상 `money` 필드가 포함되지 않도록 정리했다.
- 단, 구형 저장 데이터 호환을 위해 마이그레이션 내부에서는 `saved.money`를 읽을 수 있게 유지했다.

### 4. 신규 게임 기본 루블 지급 변경

- 신규 게임 시작 루블을 `500,000`으로 변경했다.
- 수량 없는 일반 루블 아이템을 시드로 넣는 방식은 제거했다.
- 신규 루블은 반드시 수량이 있는 스택으로 생성한다.

```js
const STARTING_ROUBLES = 500000;
createRoubleStack(STARTING_ROUBLES)
```

- `STARTING_MONEY` 이름은 구형 숫자 자금 코드와 혼동되지 않도록 제거했다.

### 5. 루블 스택 생성 및 정리 함수 통일

- 수량 있는 루블 생성은 `createRoubleStack(quantity)`로 통일했다.
- `cloneItem(getTemplate('ruble'))`을 직접 호출해서 수량 없는 루블을 만드는 경로를 제거했다.
- 루팅 테이블에서 루블이 나올 때도 수량이 있는 스택으로 생성되도록 정리했다.
- 저장 전 빈 스택과 비정상 수량은 정리된다.

```js
createRoubleStack(quantity)
cleanupEmptyStacks()
```

- 정리 대상:
  - `quantity <= 0`인 루블 스택
  - 음수 수량
  - 소수점 수량
  - 최대 스택 초과 수량

### 6. 컨테이너 자기 참조 삽입 버그 수정

- 기존에는 자기 자신을 직접 자기 내부에 넣는 것은 막았지만, 자기 내부에 있는 중첩 컨테이너에 자신을 넣는 경로가 남아 있었다.
- 이 경우 컨테이너 계층이 순환되면서 아이템이 사라질 수 있어, 중첩 컨테이너 경로까지 검사하도록 수정했다.

- 수정된 규칙:
  - 아이템은 자기 자신 내부에 들어갈 수 없다.
  - 아이템은 자기 자식 컨테이너 내부에도 들어갈 수 없다.
  - 중첩 컨테이너 드롭, 빠른 이동, 컨테이너 삽입 경로에서 같은 규칙을 적용한다.

### 7. M4A1 스코프 비조준 렌즈 분리

- M4A1 스코프 화면 재질을 조준용과 비조준용으로 분리했다.

```js
scopeLiveMaterial
scopeIdleMaterial
```

- 조준 상태에서는 기존처럼 `scopeRenderTarget.texture`를 사용한다.
- 비조준 상태에서는 실시간 화면 대신 어두운 푸른빛 렌즈 재질을 표시한다.
- 비조준 상태에서는 레티클과 빨간 점을 숨긴다.
- 조준하면 레티클과 빨간 점이 다시 표시된다.

### 8. M4A1 스코프 렌더링 최적화

- M4A1을 들고 있어도 비조준 상태에서는 스코프 카메라 렌더링을 실행하지 않도록 변경했다.

```js
const shouldRenderScope =
  currentWeapon === WEAPONS.M4A1 &&
  isAiming &&
  !deathState.active &&
  !raidEnded;
```

- `renderer.setRenderTarget(scopeRenderTarget)`와 `renderer.render(scene, scopeCamera)`는 `shouldRenderScope`가 true일 때만 실행된다.
- 사망 연출 중이거나 레이드 종료 상태에서도 스코프 렌더링이 실행되지 않는다.
- 일반 메인 카메라 렌더링은 기존처럼 마지막에 실행된다.

### 9. M4A1 스코프 렌더타겟 및 렌즈 재질 경량화

- 스코프 렌더타겟 해상도를 `1024 x 1024`에서 `512 x 512`로 낮췄다.

```js
const renderTargetWidth = 512;
const renderTargetHeight = 512;
```

- 스코프 유리 표현에 사용하던 고비용 `MeshPhysicalMaterial`을 제거했다.
- `transmission`, `thickness` 같은 고급 유리 굴절 옵션을 사용하지 않는다.
- 렌즈 표현은 `MeshBasicMaterial` 기반의 어두운 푸른빛 투명 재질로 대체했다.

### 10. FPS 무기 모델 실측 스케일 정규화

- FPS 레이드 무기 모델의 월드 단위 기준을 정리했다.

```text
1 unit = 1 meter
```

- M4A1과 MP-153 모델의 전후방 길이를 실제 치수 기준으로 정규화했다.

```text
M4A1  : 838mm  -> 0.838 unit
MP-153: 1250mm -> 1.25 unit
```

- 모델은 local X축 기준으로 길이를 측정한다.
- 모델 조립 후 bounding box의 `size.x`를 측정해 목표 길이에 맞게 최종 스케일을 적용한다.

```js
normalizeWeaponLength(model, targetLength)
```

- `weaponGroup.scale`에 남아 있던 공통 축소 배율을 제거하고 기본값으로 정리했다.

```js
weaponGroup.scale.set(1, 1, 1);
```

- 이번 패치에서는 조준 위치, 카메라 위치, 반동, 사격, 탄약, 스코프 로직은 변경하지 않았다.

## 참고

- 오늘 패치 이후 실제 자금 원본은 숫자 상태가 아니라 루블 아이템 스택이다.
- 구형 저장 데이터의 `money` 값은 마이그레이션 단계에서만 읽는다.
- 마이그레이션 완료 후 새 저장 데이터에는 `money` 필드가 남지 않는다.
- 무기 실측 스케일 패치는 화면에 보이는 모델 크기만 정리했으며, 전투 밸런스 값은 변경하지 않았다.
