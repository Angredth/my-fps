# 6.20 Patch

작성일: 2026-06-20

## 오늘 Codex로 처리한 작업

### 1. 탄환 자국 크기 축소

- FPS 레이드에서 생성되는 탄환 자국 크기를 기존의 1/3로 줄였다.
- 벽, 나무, 바닥 충돌 자국이 모두 같은 배율을 사용하도록 `bulletHoleScale`을 추가했다.

```js
const bulletHoleScale = 1 / 3;
```

### 2. Gun ADS 튜너와 SKOV POSE 연동

- `Gun-ads.html`에서 조정한 무기 POSE를 `SKOV.html`에 바로 적용할 수 있도록 `localStorage` 기반 브리지를 추가했다.
- `Apply to SKOV` 버튼을 누르면 다음 값이 저장된다.
  - `WEAPON_VIEW_POSES`
  - `M4_SCOPE_POSE`
  - Q/E lean 조정값
- `SKOV.html`은 시작 시 저장된 값을 읽고, 이미 열린 상태에서는 `storage` 이벤트로 즉시 반영한다.

```js
const POSE_BRIDGE_STORAGE_KEY = 'skov.weaponPose.v1';
```

### 3. CDN 무기 모델 로딩 추가

- FN 5-7, ER Fulcrum 근접무기, AKS-74U 모델을 CDN에서 불러오도록 정리했다.

```text
https://cdn.jsdelivr.net/gh/Angredth/my-fps@main/5.28/index/57.js
https://cdn.jsdelivr.net/gh/Angredth/my-fps@main/5.28/index/er-fulcrum.js
https://cdn.jsdelivr.net/gh/Angredth/my-fps@main/5.28/index/AKS.js
```

- `57.js`와 `AKS.js`는 전역 factory 함수를 등록하는 classic script 방식으로 로드한다.
- `er-fulcrum.js`는 ES module import 방식으로 로드한다.
- CDN 로드 또는 모델 생성에 실패하면 기존 fallback 모델을 사용한다.

### 4. FN57, 근접무기, AKS-74U 실측 스케일 정리

- `1 unit = 1m` 기준으로 무기 모델 길이를 정리했다.

```text
FN 5-7     : 355mm -> 0.355 unit
ER Fulcrum : 310mm -> 0.310 unit
AKS-74U    : 730mm -> 0.730 unit
```

- FN57과 ER Fulcrum은 원본 모델 길이 기준으로 스케일을 계산한다.

```js
const FN57_MODEL_SCALE = 0.355 / 9.285;
const KNIFE_MODEL_SCALE = 0.310 / 11.679;
```

- AKS-74U는 원본 모델의 앞뒤 축인 local X축을 먼저 `0.730`으로 정규화한 뒤 화면 방향 회전을 적용한다.
- 회전 후 bounding box 기준으로 정규화하면 실제 전후방 길이가 틀어질 수 있어 순서를 조정했다.

### 5. 모델 중앙 pivot 정렬

- `Gun-ads.html`과 `SKOV.html`에서 FN57, ER Fulcrum 모델을 중앙 pivot 그룹으로 감싸도록 변경했다.
- POSE 회전 슬라이더가 모델 offset 기준이 아니라 모델 중심 기준으로 동작한다.
- FN57 총구 마커와 근접무기 타격 마커는 모델 child로 먼저 붙인 뒤 pivot으로 감싸서, 발사 및 타격 위치가 모델 보정과 함께 따라가도록 했다.

```js
centerModelOnPivot(model, pivotName)
```

### 6. Gun ADS 튜너 개선

- `Gun-ads.html`에 AKS-74U를 4번 무기로 추가했다.
- 4번 버튼과 `Digit4` 키바인딩으로 AKS-74U를 선택할 수 있다.
- Q/E lean을 누르고 있는 방식에서 토글 방식으로 변경했다.
  - `Q`: 왼쪽 lean 토글
  - `E`: 오른쪽 lean 토글
  - 반대 방향 입력 시 기존 lean 해제 후 반대 방향으로 전환
  - 창 포커스를 잃으면 lean 해제
- Q/E lean 수치 조정 UI를 추가했다.
  - 카메라 좌우 이동
  - 카메라 하강
  - 카메라 roll 각도
  - lean 중 무기 위치 보정
  - lean 중 무기 yaw/roll 보정
  - ADS 중 무기 lean 보정 축소 비율

### 7. SKOV lean 설정 연동

- `SKOV.html`의 Q/E lean 하드코딩 값을 `leanSettings` 객체로 정리했다.
- `Gun-ads.html`에서 조정한 lean 수치가 `Apply to SKOV`를 통해 실제 게임 lean에도 반영된다.

```js
const leanSettings = {
  cameraSide: 0.42,
  cameraDrop: 0.08,
  cameraRollDeg: 14,
  weaponSide: 0.055,
  weaponDrop: 0.025,
  weaponForward: 0.02,
  weaponYaw: 0.025,
  weaponRoll: 0.08,
  adsWeaponScale: 0.2
};
```

### 8. M4A1 POSE 및 스코프 감도 정리

- `Gun-ads.html`의 M4A1 기본 hip/ADS POSE를 `SKOV.html` 기준값으로 맞췄다.
- `Gun-ads.html`에서 ADS 전환 시 카메라 FOV를 60으로 변경하던 동작을 제거하고, SKOV와 같이 75를 유지하도록 했다.
- M4A1 스코프 배율 표시 체계를 고려해, `16x` 표시 상태에서만 마우스 감도를 절반으로 줄였다.
- `4x` 표시 상태는 실제 1배 줌 체감이므로 기본 감도를 유지한다.

```js
const BASE_LOOK_SENSITIVITY = 0.002;
const M4_SCOPE_16X_LOOK_SENSITIVITY_MULTIPLIER = 0.5;
```

## 참고

- `Gun-ads.html`의 `Apply to SKOV`는 같은 origin에서 열린 `SKOV.html`에 적용된다.
- 이전 POSE가 `localStorage`에 남아 있으면, 새 조정값을 반영하기 위해 `Apply to SKOV`를 다시 눌러야 한다.
- CDN 모델은 GitHub main 브랜치의 파일 상태에 의존한다.
