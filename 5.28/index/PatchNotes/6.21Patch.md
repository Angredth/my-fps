# 6.21 Patch

작성일: 2026-06-21

## 오늘 Codex로 처리한 작업

### 1. 총기 이동 모션 강도 재조정

- 이동 시 총기 흔들림이 과하게 보이던 문제를 줄였다.
- 변경된 총기 크기에 맞춰 무기별 이동 모션 배율을 다시 맞췄다.
- 걷기/질주 상태에서 적용되는 위치, 회전 오프셋 강도를 전체적으로 낮췄다.

### 2. 인벤토리 드래그 기준을 아이템 중심으로 통일

- 여러 칸 아이템을 드래그할 때, 마우스가 닿은 칸을 좌측 상단으로 보던 기존 기준을 제거했다.
- 이제 드래그 고스트, 배치 미리보기, 실제 드롭 좌표가 모두 아이템 중심 기준으로 계산된다.
- 회전된 아이템도 현재 크기 기준으로 중심 오프셋을 다시 계산하도록 정리했다.

적용 규칙:

```txt
마우스 포인터 위치 = 드래그 고스트 중심 = 배치 미리보기 중심
```

추가된 핵심 함수:

```js
getItemCenterGridOffset(item)
getCenteredGridDropPosition(item, hoveredX, hoveredY)
```

### 3. 드래그 고스트와 드래그 시작 안정화

- `setDragImage()`를 사용해 드래그 고스트가 항상 아이템 중앙 기준으로 따라오도록 수정했다.
- 드래그 시작 직후 `.dragging` 클래스가 네이티브 드래그를 방해하던 문제를 줄이기 위해, 실제 드래그가 시작된 뒤에만 드래그 스타일을 붙이도록 바꿨다.
- 드래그 중 아이템 DOM이 슬롯 포인터 이벤트를 막지 않도록 `pointer-events: none`을 유지했다.

### 4. 상점 거래 패널 상단 바 개편

- 거래 패널 상단을 다음 구조로 교체했다.
  - 좌측: `필요 항목 자동 넣기` 토글
  - 중앙: `구매 / 판매` 탭
  - 우측: `판매 등록 취소`
- 판매 등록 아이템이 없을 때 상단 취소 버튼이 비활성화되도록 연결했다.
- 기존 판매 패널 하단의 `등록 취소` 버튼은 제거했다.

### 5. 구매/판매 공통 `거래하기` 버튼 통합

- 구매와 판매가 서로 다른 버튼을 쓰던 구조를 공통 `거래하기` 버튼으로 통일했다.
- 버튼 우측에는 현재 거래 금액이 표시된다.
  - 구매 모드: 총 구매 금액
  - 판매 모드: 총 판매 수익
- 거래가 불가능한 상태에서는 버튼이 비활성화된다.

비활성 조건 예시:

- 선택한 구매 아이템 없음
- 레벨 제한 미달
- 루블 부족
- 스태시 공간 부족
- 판매 등록 아이템 없음
- 판매 수익을 받을 공간 부족

### 6. 구매 패널을 Tarkov식 흐름 레이아웃으로 변경

- 구매 모드 본문을 상하 2단 흐름 구조로 재배치했다.
  - `구매할 아이템: [선택 아이템명]`
  - `구매하기위해 필요한 아이템:`
- 두 제목은 같은 중심축 기준으로 정렬되도록 바꿨다.
- 좌우 화살표는 아이템 미리보기 쪽에 더 가깝게 붙도록 간격을 조정했다.

### 7. 구매 수량 입력 지원

- 구매할 아이템 아이콘 우측에 `x [수량 입력]` 레이아웃을 추가했다.
- 입력한 수량만큼 실제 구매가 가능하도록 `buyItem()`을 확장했다.
- 총 구매 금액, 필요 루블 표시, 구매 가능 여부 판정이 모두 입력 수량을 기준으로 갱신된다.
- 입력값 자릿수에 맞춰 수량 입력 칸의 좌우 폭이 자동으로 늘어나도록 했다.

구매 수량 입력 규칙:

- 숫자만 입력 가능
- 포커스 시 전체 선택
- 비어 있는 상태로 편집 가능
- 포커스 해제 시 비어 있으면 `1`로 강제 복귀
- 별도 증가/감소 다이얼 제거

### 8. 필요 아이템 수치 표시 정리

- `구매하기위해 필요한 아이템` 영역에서 루블 아이콘 우측 하단에 수치가 표시되도록 변경했다.
- 표기 형식은 다음으로 통일했다.

```txt
[현재 거래 금액]/[필요 금액]
```

예:

```txt
3200/3200
```

- 좌측 값은 현재 입력한 구매 수량 기준 총 거래 금액과 동일하다.
- 루블이 부족하면 부족 상태 색상과 강조가 적용된다.

### 9. 분할 입력창도 동일한 숫자 입력 방식으로 통일

- 스택 분할 모달의 수량 입력 칸에도 같은 입력 규칙을 적용했다.
- 숫자만 입력 가능하며, 포커스 시 전체 선택되고, 입력 자릿수에 맞춰 폭이 바뀐다.
- 비워 둔 상태로 편집할 수 있지만, 포커스를 잃으면 `1`로 복구된다.
- 기존 수치 조절 다이얼은 제거했다.

### 10. 검증

- `SKOV.html`의 inline script 2개를 분리해 문법 체크를 진행했다.
- `node --check` 기준으로 스크립트 문법 오류가 없는 상태를 확인했다.
- 브라우저 실동작은 이번 패치노트 작성 시점에 별도 재검증하지 않았다.
