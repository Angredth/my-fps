# 6.24 Patch

작성일: 2026-06-24

## 오늘 Codex로 처리한 작업

### 1. 상점 판매창 레이아웃 정리

- 판매 모드의 기존 안내 문구를 제거했다.
- 판매창 하단의 기존 `판매하기`, `등록 취소` 버튼 레이아웃을 제거했다.
- 판매 확정은 상단 공통 `거래하기` 버튼에서 처리하도록 정리했다.
- 판매 등록 취소는 상단 우측 취소 버튼에서 처리하도록 유지했다.
- 판매 모드 본문을 `trade-sell-layout` 구조로 분리했다.

### 2. 판매 그리드 기본 크기 변경

- 판매 그리드 기본 크기를 `8x11`로 변경했다.
- 판매 그리드 전용 스타일을 추가해 상점 거래 패널 안에서 더 어두운 Tarkov식 그리드로 보이도록 했다.
- 판매 그리드는 거래 패널 중앙에 정렬되도록 수정했다.

```js
const SELL_COLS = 8;
const SELL_BASE_ROWS = 11;
```

### 3. 판매 그리드 자동 세로 확장

- 판매 영역이 가득 찼을 때 세로 행을 자동으로 늘리도록 구현했다.
- 확장은 4행 단위로 진행되며, 최대 60행까지만 늘어난다.
- 판매 그리드가 커지면 판매 영역 내부에서 세로 스크롤로 확인할 수 있다.
- 판매 아이템이 남아 있는 동안에는 자동으로 행을 줄이지 않는다.

```js
const SELL_ROW_GROW_STEP = 4;
const SELL_MAX_ROWS = 60;
```

추가된 핵심 함수:

```js
getSellRows()
findSellSpotWithAutoGrow(item)
```

초기화 조건:

- 판매 완료 후 `trade.sellRows`를 기본 행 수로 복구
- 등록 취소로 판매 아이템이 모두 돌아간 경우 기본 행 수로 복구
- 스태시 공간 부족으로 등록 취소가 중간 실패하면 남은 판매 아이템과 확장 행 수 유지

### 4. 상점 그리드 크기와 정렬 조정

- 전체 인벤토리 그리드 슬롯 기준을 `50px`로 키웠다.
- 판매 그리드와 상점 스태시 그리드가 같은 슬롯 크기 체계를 사용하도록 정리했다.
- 상점 3열 레이아웃 폭을 키워 커진 판매/스태시 그리드가 잘리지 않도록 했다.
- 상점 우측 스태시 그리드는 우측 정렬되도록 `trader-stash-grid-shell`을 추가했다.
- 아이템 카드 내부 이미지 영역도 슬롯 크기에 맞춰 커지도록 조정했다.

```css
--grid-slot: 50px;
--grid-gap: 3px;
```

### 5. POCKETS UI 정리

- `Suck/Pokets.png`를 참고해 `POCKETS` 표시 헤더를 짧은 Tarkov식 라벨 형태로 정리했다.
- 기존의 긴 설명형 문구를 제거하고 `POCKETS` 표기로 통일했다.
- 포켓 그리드는 4칸 전체가 스크롤 없이 보이도록 유지했다.
- 일반 인벤토리, 레이드 장비, 루팅 오버레이, 시체 루팅 포켓에 같은 스타일을 적용했다.

적용 범위:

- 일반 인벤토리 `pockets`
- 레이드 플레이어 장비 `pockets`
- 레이드 루팅 오버레이 `POCKETS`
- 시체 루팅 `POCKETS`

### 6. FPS 레이드 루팅 수색 후 상호작용 먹통 버그 수정

- FPS 레이드에서 루팅 테이블 수색 완료 후 예전 텍스트/2D 레이드용 위험 이벤트가 발동되던 호출을 제거했다.
- `completeSearch()`에서 더 이상 `eventChance()`와 `triggerRiskEvent()`를 호출하지 않는다.
- 수색 완료 후 `raid.pendingEvent`가 생성되어 포인터락 복귀 후 루팅 테이블/시체 상호작용이 먹통 되는 흐름을 차단했다.
- 이미 런타임에 남아 있는 `raid.pendingEvent`가 있을 경우, 다음 월드 루팅 상호작용 시 자동으로 정리하도록 `openWorldLootContainer()`에 방어 코드를 추가했다.
- `triggerRiskEvent()`, `eventChance()`, `resolveEvent()` 함수 자체는 삭제하지 않고, FPS 루팅 수색 완료 경로에서만 호출되지 않도록 정리했다.

변경된 흐름:

```js
// completeSearch()
render();
```

```js
// openWorldLootContainer()
if (raid.pendingEvent) {
  console.warn('루팅 상호작용을 막고 있던 pendingEvent를 정리했습니다.', raid.pendingEvent);
  raid.pendingEvent = null;
}
```
