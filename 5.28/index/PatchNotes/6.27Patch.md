# 6.27 Patch

작성일: 2026-06-27

## 오늘 Codex로 처리한 작업

### 1. AKS-74U 무기 추가

- 신규 주무기 `AKS-74U`를 아이템 카탈로그에 추가했다.
- 크기는 `4x2`, 장착 타입은 `primary`로 설정해 주무기 1/2 슬롯에 장착 가능하도록 했다.
- 상점 판매 목록에 `45,000₽`, 재고 `3/3` 무기로 추가했다.
- FPS 레이드 무기 키 `AKS74U`를 추가했다.
- AKS-74U는 기존 자동화기 사격 루프를 공유하며, M4A1과 같은 방식으로 연사 처리된다.
- AKS-74U는 스캐브 무기 후보에 포함되며, AKS-74U를 든 스캐브 처치 시 시체 루팅 패널에 같은 아이템이 드랍된다.

### 2. DVL-10 무기 추가

- 신규 주무기 `DVL-10`을 아이템 카탈로그에 추가했다.
- 크기는 `5x2`, 장착 타입은 `primary`로 설정해 주무기 1/2 슬롯에 장착 가능하도록 했다.
- 상점 판매 목록에 `135,000₽`, 재고 `1/1` 무기로 추가했다.
- FPS 레이드 무기 키 `DVL10`을 추가했다.
- DVL-10 전용 포즈와 이동 모션 배율을 추가했다.
- DVL-10은 스캐브 무기 후보에 넣지 않아 적 장비로 스폰되지 않도록 했다.

### 3. DVL-10 볼트액션 입력 흐름 재작업

- DVL-10 발사 성공 직후 `chambered=false`, `needsBolt=true` 상태가 되도록 정리했다.
- 좌클릭을 계속 누르고 있는 동안에는 볼트액션이 실행되지 않도록 막았다.
- 좌클릭을 뗀 순간 `needsBolt && !cyclingBolt` 상태라면 자동으로 볼트액션을 시작한다.
- 볼트액션 중에는 `shootWeapon()`이 실행되지 않도록 차단했다.
- 볼트액션 완료 후 `chambered=true`, `needsBolt=false`, `cyclingBolt=false`가 되어 다시 사격 가능하도록 했다.
- R키로 볼트액션을 실행하는 독립 바인딩은 제거했다.
- 기존 R키 재장전/인벤토리/아이템 회전 흐름은 유지했다.
- 자동화기, 샷건, 권총, 근접무기의 기존 사격 흐름은 유지했다.

### 4. DVL-10 볼트액션 모션 개선

- `Gun-ads.html`의 DVL-10 볼트액션 상태/모션 구조를 `SKOV.html`에 맞게 이식했다.
- `DVL10_BOLT_ACTION`, `DVL10_BOLT_PIVOT`, `DVL10_ACTION_POSE` 기반의 볼트 피벗/슬라이드/회전 모션을 적용했다.
- 단순 왕복 이동이 아니라 볼트 들어올림, 뒤로 당김, 앞으로 밈, 내림 단계가 진행되도록 했다.
- 볼트 조작 중 총 전체에 작게 흔들리는 액션 포즈 오프셋을 추가했다.
- 볼트 메쉬를 찾지 못해도 게임이 멈추지 않도록 `console.warn` 후 상태 로직만 진행하게 했다.
- 로컬 DVL-10 모델 경로를 실제 존재하는 `dvl-10-edited (1).glb`로 맞춰 초기 404 콘솔 오류를 제거했다.

### 5. DVL-10 탄약 상태 정리

- DVL-10은 `약실 1발 + 탄창 9발` 구조로 동작한다.
- 레이드 시작 시 `10/10 READY` 상태로 초기화된다.
- 발사 후에는 약실이 비고 `NEED BOLT` 상태가 된다.
- 볼트액션 완료 시 탄창 잔탄 1발을 약실로 올리고, 탄창 잔탄을 1 감소시킨다.
- 재장전 중과 볼트액션 중에는 발사가 차단된다.
- 무기 교체 시 진행 중인 DVL-10 볼트/재장전 동작은 취소하되, 탄약 수 자체는 초기화하지 않도록 했다.

### 6. DVL-10 격발음 적용

- DVL-10 전용 격발음 `DVLR.wav`를 추가했다.
- 적용 소스:

```text
https://raw.githubusercontent.com/Angredth/my-fps/main/GunFx/DVL10/DVLR.wav
```

- 기존 M4A1/AKS-74U/MP-153/FN57 사운드와 분리된 DVL-10 전용 샘플 풀을 사용한다.
- `Sound.playGunshot('DVL-10')` 호출 시 DVL-10 격발음이 우선 재생된다.
- DVL-10 격발음 끝부분은 페이드 아웃 처리했다.
- 페이드 설정은 `0.32초`부터 `0.62초` 동안 볼륨을 0까지 낮추는 방식이다.

### 7. DVL-10 볼트액션 사운드 추가

- DVL-10 볼트액션 전용 사운드 `Bolt.wav`를 추가했다.
- 적용 소스:

```text
https://raw.githubusercontent.com/Angredth/my-fps/main/GunFx/DVL10/Bolt.wav
```

- `DVLR.wav` 격발음과 별개로 볼트액션 모션 시작 시점에만 1회 재생된다.
- 매 프레임 반복 재생되지 않도록 `startDVL10BoltCycle()` 진입 성공 시점에만 호출한다.
- `DVL10_BOLT_ACTION.duration` 값에 맞춰 자동으로 재생속도를 보정한다.
- 현재 기준 `Bolt.wav` 길이 `0.942625초`, 모션 길이 `0.95초`라서 재생속도는 약 `0.9922x`로 계산된다.
- 나중에 볼트액션 모션 길이를 바꿔도 사운드 속도가 자동으로 따라가도록 했다.

### 8. DVL-10 소음기 판정

- DVL-10은 소음기 장착 총으로 취급되도록 적의 소리 인식 범위를 줄였다.
- 일반 총기 사격 소음 반경은 기존 `70`을 유지했다.
- DVL-10 사격 소음 반경은 `22`로 별도 설정했다.
- M4A1, AKS-74U, MP-153, FN57의 적 소리 인식 반경은 변경하지 않았다.

```js
hearingDistanceGunshot: 70,
hearingDistanceSuppressedGunshot: 22
```

### 9. FPS 모델 및 fallback 안정화

- AKS-74U와 DVL-10 모델 생성은 `createSafeWeaponModel()`을 거쳐 실패 시 fallback 또는 빈 `THREE.Group`으로 대체되도록 했다.
- AKS-74U와 DVL-10에 각각 머즐 마커를 추가해 기존 탄환/머즐 플래시 경로를 사용하도록 했다.
- `WEAPONS` 레지스트리를 검증해 누락된 mesh/marker를 콘솔에 표시하도록 했다.
- `applyWeaponLoadout()`와 `triggerWeaponSwap()`에서 잘못된 무기 키나 mesh 누락이 전체 렌더 중단으로 이어지지 않게 방어했다.

### 10. FPS 레이드 검은화면 방어 패치

- `startRaid()`에서 `window.FPSRaid?.start()`로 조용히 실패하던 구조를 제거했다.
- FPS 모듈이 없거나 `start()`가 준비되지 않은 경우 콘솔 오류와 토스트를 띄우고 메뉴로 즉시 복귀하도록 했다.
- FPS 모듈 준비 완료 시 `window.FPSRaidReady = true`, `FPSRaidReady` 이벤트, `[FPSRaid] ready` 로그를 남기도록 했다.
- `FPSRaid.start()`와 `animate()`에 오류 표시용 `try/catch`를 추가해 검은화면 대신 blocker/weapon UI에 실패 원인을 표시하도록 했다.

### 11. 상점/아이템 UI 및 루팅 관련 유지 작업

- 상점 판매창, 판매 그리드, POCKETS UI의 기존 정리 작업을 유지했다.
