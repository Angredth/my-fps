# 6.28 Patch

작성일: 2026-06-28

## 오늘 Codex로 처리한 작업

### 1. 아이템 이동 Grab/Drop 효과음 추가

- 인벤토리 아이템 이동용 `itemMoveSfxSources`를 추가했다.
- 방어구, 헬멧, 리그, 탄약, 소음기, 탄창, 주요 총기, 의료 아이템, 열쇠, 지갑에 각각 Grab/Drop 사운드 키를 매핑했다.
- 모든 신규 FX 경로는 `cdn.jsdelivr.net/gh/Angredth/my-fps@main/5.28/index/FX/...` 형식으로 통일했다.
- GitHub `blob` 링크를 `Audio.src`에 직접 넣지 않도록 정리했다.

### 2. 인터페이스 원샷 효과음 재생 함수 추가

- `playOneShotInterfaceSfx()`를 추가해 UI/아이템 효과음을 공통 방식으로 재생하도록 했다.
- 새 효과음은 기존 설정창의 `master`, `fx` 볼륨을 따르는 `interface` 카테고리로 재생된다.
- 레이드 결과음 재생 중에는 신규 인터페이스 효과음이 끼어들지 않도록 `raidOutcomeSoundPlaying` 체크를 유지했다.
- 재생 실패 시 다음 소스로 fallback하는 기존 버튼 효과음 계열의 패턴을 유지했다.

추가된 핵심 함수:

```js
playOneShotInterfaceSfx(sources, baseVolume)
getItemSfxKey(item)
playItemMoveSfx(item, phase)
playInventoryUiSfx(kind)
showErrorToast(message)
```

### 3. Grab 재생 위치 정리

- 일반 인벤토리 그리드 아이템은 `renderGrid()`의 실제 `dragstart`에서만 Grab이 재생되도록 했다.
- 장착 슬롯 아이템은 `createEquipSlotNode()`의 실제 `dragstart`에서만 Grab이 재생되도록 했다.
- 시체 장비 아이템은 `createCorpseItemNode()`의 실제 `dragstart`에서만 Grab이 재생되도록 했다.
- 클릭, 더블클릭, Ctrl+클릭 빠른 이동에서는 Grab이 재생되지 않는다.

### 4. Drop 재생 조건 정리

- DOM `drop` 이벤트에서 바로 Drop을 재생하지 않고, 실제 이동 성공 함수 내부에서만 재생하도록 했다.
- 같은 그리드의 같은 좌표로 되돌아간 경우에는 Drop을 재생하지 않는다.
- 실패 드롭, 배치 불가, 컨테이너 제한, 원위치 복귀에서는 Drop을 재생하지 않는다.
- Ctrl+클릭 빠른 이동은 내부 성공 경로에서 Drop이 1회만 재생되도록 했다.

Drop 적용 성공 경로:

- `moveBetweenAreas()`
- `equipFromArea()`
- `equipToCorpseSlot()`
- `insertIntoContainer()`
- `mergeStackIntoTarget()`
- `confirmStackSplit()`
- `registerSaleFromStash()`
- `clearSaleGrid()`
- `attachSuppressor()`

### 5. 스태쉬/리그 UI 효과음 추가

- 레이드 중 Tab 인벤토리 열기 시 `Stashopen.wav`가 재생되도록 했다.
- 레이드 인벤토리, 수색, 루팅 컨테이너, 시체 루팅 오버레이가 실제로 닫힐 때 `Stashout.wav`가 재생되도록 했다.
- 메인/타이틀 인벤토리 화면을 열고 닫을 때도 스태쉬 열기/닫기 효과음을 적용했다.
- 리그 컨테이너를 새로 열 때 `Rigopen.wav`가 재생되도록 했다.
- 리그 컨테이너를 더블클릭 토글 또는 닫기 버튼으로 닫을 때는 신규 소스가 아니라 기존 `Esc.mp3` 구조를 사용하도록 했다.

### 6. 상점 거래 성공 효과음 추가

- 구매/판매 확정 버튼에 `data-trade-execute` 식별자를 추가해 전역 버튼 클릭음 `Botton.mp3`가 먼저 재생되지 않도록 했다.
- 구매 성공 시 `Trade.wav`가 1회 재생되도록 했다.
- 판매 성공 시 `Trade.wav`가 1회 재생되도록 했다.
- 루블 부족, 공간 부족, 판매 불가 등 실패한 거래에서는 `Trade.wav`가 재생되지 않는다.

### 7. 실패 메시지 효과음 분리

- `showToast()` 자체에는 효과음을 넣지 않았다.
- 실패성 메시지만 `showErrorToast()`를 거치도록 분리해 `Zzz.wav`가 재생되도록 했다.
- 장착 성공, 판매 등록, 구매 완료, 판매 완료, 회전, 소음기 장착 성공 같은 성공/상태 메시지는 기존 `showToast()`를 유지했다.

오류음 적용 대표 메시지:

- 빠른 이동할 공간이 없습니다.
- 공간 또는 컨테이너 제한 때문에 배치할 수 없습니다.
- 컨테이너 내부에 넣을 수 없습니다.
- 장착 실패: 슬롯 타입이 맞지 않거나 슬롯이 비어 있지 않습니다.
- 장착 실패: 시체 장비 슬롯 타입이 맞지 않거나 슬롯이 비어 있지 않습니다.
- 공간 부족
- 루블이 부족합니다.
- 판매 영역 공간이 부족합니다.
- 판매 수익을 보관할 스태시 공간이 부족합니다.
- 스태시 공간이 부족해 판매 등록을 취소할 수 없습니다.

### 8. 중복 재생 방지

- 거래 실행 버튼은 기본 버튼 클릭음을 건너뛰고 성공 함수에서만 `Trade.wav`를 재생한다.
- 리그 닫기 버튼은 기본 버튼 클릭음을 건너뛰고 기존 `Esc.mp3`만 재생한다.
- Esc로 인벤토리를 닫는 경우 기존 `Esc.mp3`와 `Stashout.wav`가 겹치지 않도록 처리했다.
- 스택 병합 후 남은 아이템을 이어서 이동하는 빠른 이동 경로는 Drop이 중복 재생되지 않도록 `playDropSfx` 옵션으로 제어했다.

### 9. 유지된 동작

- 기존 버튼 hover/click/escape 효과음 구조는 유지했다.
- BGM과 레이드 결과음 구조는 변경하지 않았다.
- 기존 사운드 설정의 `master`, `fx`, `music` 구조는 변경하지 않았다.
- 새 UI/아이템 효과음은 모두 `interface` 볼륨 경로를 사용한다.

### 10. DVL-10 아이콘 제작 워크벤치 추가

- `DVL10IconWorkbench.html` 단일 HTML 툴을 추가했다.
- `SKOV.html`의 DVL-10 모델 로딩 구조를 참고하되, 게임 인벤토리 로직과 분리된 독립 툴로 구성했다.
- 로컬 GLB 파일 선택 기능을 추가해 사용자가 직접 DVL-10 모델 파일을 우선 로딩할 수 있도록 했다.
- 로컬 파일이 없으면 기존 DVL-10 GLB URL 목록을 순서대로 시도하고, 전부 실패하면 fallback DVL-10 모델을 표시하도록 했다.
- OrthographicCamera 기반으로 인벤토리 아이콘용 투명 PNG를 렌더링하도록 했다.
- 모델 회전, 위치, 스케일, 카메라 줌, 조명 강도, 배경 미리보기 모드를 조정할 수 있도록 했다.
- `5x2` 인벤토리 카드 미리보기를 추가해 DVL-10이 실제 카드 비율에서 어떻게 보이는지 확인할 수 있도록 했다.
- `320x128`, `640x256`, `1024x410` PNG 출력 옵션을 제공한다.
- 현재 조정값을 JSON 프리셋으로 복사할 수 있도록 했다.
- `ITEM_CATALOG`에 넣을 icon 패치 코드를 복사할 수 있도록 했지만, 실제 `SKOV.html` 인벤토리 적용은 이번 최종 패치에 포함하지 않았다.
